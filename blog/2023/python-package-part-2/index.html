Rootless Podman with Python and Jekyll

April 23, 2023

2023   ·   Python   Jekyll   Podman     ·   Cross Python Development  

In this post I’ll describe a Podman Containerfile which describes how to set up a Python development environment with rootless Podman. It is based on a Gist on GitHub published by “BrutalSimplicity” and Dane Hillard’s book Publishing Python Packages. Furthermore, it contains a full Ruby installation (necessary for creating Jekyll static webpages and blog posts, and GitHub Pages). Thus,

FROM debian:bullseye

# note: use with care
ARG DEBIAN_FRONTEND=noninteractive

LABEL maintainer="Carl Johan Rehn <care02@gmail.com>"
LABEL updated_at=2023-03-06

RUN apt-get update -y

Add locale and a time zone (check in running container with $TZ, date, and locale), see question on stackoverflow and Jiménez’s Gist on GitHub

RUN apt-get install -y locales locales-all tzdata

RUN sed -i '/en_US.UTF-8/s/^# //g' /etc/locale.gen && \
    locale-gen
ENV LANG en_US.UTF-8
ENV LANGUAGE en_US:en
ENV LC_ALL en_US.UTF-8
# ENV TZ="America/New_York"

For local language settings, change accordingly

RUN apt-get install --no-install-recommends ruby-full build-essential zlib1g-dev -y

Clean up

RUN apt-get clean && \
    rm -rf /var/lib/apt/lists/

Use bash for all RUN commands

SHELL ["/bin/bash", "-lc"]

Install python-launcher (requires root)

RUN curl --location --remote-name https://github.com/brettcannon/python-launcher/releases/download/v1.0.0/python_launcher-1.0.0-x86_64-unknown-linux-gnu.tar.xz && \
    tar --extract --strip-components 1 --directory /usr/local --file python_launcher-1.0.0-x86_64-unknown-linux-gnu.tar.xz && \
    rm python_launcher-1.0.0-x86_64-unknown-linux-gnu.tar.xz

For hints on how to set up a rootless Podman image, see Python Speed and questions on stackoverflow.

Create a non-root user and set a working directory

RUN useradd --create-home app
USER app
WORKDIR /home/app

Install asdf, The Multiple Runtime Version Manager

RUN git clone https://github.com/asdf-vm/asdf.git $HOME/.asdf --branch v0.11.1 && \
    echo ". $HOME/.asdf/asdf.sh" >> $HOME/.bashrc && \
    echo ". $HOME/.asdf/asdf.sh" >> $HOME/.profile

Add the asdf Python plugin, install python (note: asdf builds from source), and set the global Python version

RUN asdf plugin add python && \
    asdf install python 3.10.8
    asdf global python 3.10.8

Upgrade pip, the python package manager

RUN pip install --upgrade pip

Install pipx

RUN py -3.10 -m pip install pipx && \
    asdf reshim python && \
    pipx ensurepath

Install build, tox, pre-commit, and cookiecutter

RUN pipx install build && \
    pipx install tox && \
    pipx install pre-commit && \
    pipx install cookiecutter

Optional: add accompanying code from book Publishing Python Packages

RUN mkdir references && \
    git clone https://github.com/daneah/publishing-python-packages.git && \
    mv publishing-python-packages references/publishing-python-packages

Let’s build an image with podman (remember to use use the format docker flag; the no-cache flag is optional). Hence,

podman image build --no-cache --format docker -f Containerfile-pydev-jekyll -t pydev-jekyll .

Now, create a container

podman container run -it pydev-jekyll

and check that Python is available
